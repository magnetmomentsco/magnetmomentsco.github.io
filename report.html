Why these colors?

Dusty Rose (#C77D8A) is the primary brand color. It was chosen because Magnet Moments Co. is a family-run business with a warm, feminine, handcrafted feel. Dusty rose communicates approachability, creativity, and care ‚Äî without being overly "girly." It's the most common color in the original product photography (packaging, backgrounds, accents on magnets), so it creates visual continuity between the site and the products.

Sage Green (#7A9D54) provides earthy contrast. It's used for CTAs that need to stand apart from the primary buttons (like "Book an Event" vs "Shop Now"), and it evokes the natural, handmade quality of the products. Sage + dusty rose is a classic pairing in lifestyle branding ‚Äî it feels like a cozy kitchen or a well-curated fridge.

Warm Gold (#D4A574) is the accent used sparingly for badges, highlights, and hover states. It adds a subtle premium feel without the "luxury brand" coldness of pure gold. It complements the warm cream background.

Cream (#FFF8F0) replaces harsh white as the background color. Pure white (#FFF) feels clinical and stark ‚Äî cream feels warm, inviting, and "homey," which matches the brand's personality. The slight warmth also makes the product photos (which are shot in warm lighting) feel more natural.

Why we had to ditch the JS Buy SDK

üêõ Bug we discovered & fixed
We initially used Shopify's official JS Buy SDK v2 (sdks.shopifycdn.com/js-buy-sdk/v2/latest/), but it silently failed ‚Äî the "Add to Cart" button showed "Adding‚Ä¶" but the cart never populated.

Root cause: The JS Buy SDK v2 internally uses the checkoutCreate GraphQL mutation, which Shopify removed entirely from API version 2025-01. The SDK was sending mutations to a dead endpoint.

Fix: We replaced the SDK with direct fetch() calls to the Storefront GraphQL API using the modern Cart API (cartCreate, cartLinesAdd, etc.). This also eliminated the ~40KB SDK dependency ‚Äî the entire cart module is now 310 lines of vanilla JS with zero external dependencies.

What's a Content Security Policy?

A CSP is like a whitelist that tells the browser: "only load scripts, fonts, and images from these specific trusted sources." If someone somehow injected malicious code into the page, the browser would refuse to run it because it's not on the approved list.
